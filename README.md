# SpaceOps Monitoring Dashboard

## Overview

SpaceOps Monitoring Dashboard is a full-stack web application that simulates monitoring and visualization of telemetry generated by satellite ground stations.

The project was developed to demonstrate backend API development, frontend integration, database design, containerization, and deployment practices using a modern software engineering stack.

The application collects telemetry data from simulated ground stations and presents operational metrics through a centralized monitoring dashboard.

---

## Architecture

The system follows a layered client-server architecture.

```text
┌─────────────────────┐
│      Angular        │
│      Frontend       │
└──────────┬──────────┘
           │ HTTP/REST
           ▼
┌─────────────────────┐
│      FastAPI        │
│      Backend        │
└──────────┬──────────┘
           │ SQLAlchemy
           ▼
┌─────────────────────┐
│     PostgreSQL      │
│      Database       │
└──────────┬──────────┘
           ▲
           │
┌─────────────────────┐
│ Telemetry Simulator │
└─────────────────────┘
```

### Component Responsibilities

**Angular Frontend**

* Dashboard visualization
* Metrics presentation
* Station details view
* API consumption

**FastAPI Backend**

* REST API implementation
* Business logic
* Telemetry aggregation
* Health monitoring

**PostgreSQL**

* Telemetry persistence
* Station data storage
* Metrics source of truth

**Telemetry Simulator**

* Generates realistic operational data
* Updates station telemetry records
* Simulates live monitoring workloads

## Features

### Dashboard

* Ground station monitoring
* System metrics overview
* Telemetry visualization
* Operational status tracking

### Station Monitoring

Each ground station exposes:

* Station Name
* CPU Usage
* Memory Usage
* Signal Strength
* Active Connections
* Station Status
* Last Telemetry Timestamp

### API Endpoints

| Method | Endpoint                | Description                |
| ------ | ----------------------- | -------------------------- |
| GET    | `/api/v1/stations`      | Retrieve all stations      |
| GET    | `/api/v1/stations/{id}` | Retrieve station details   |
| GET    | `/api/v1/metrics`       | Retrieve dashboard metrics |
| GET    | `/health`               | Service health check       |

## Design Considerations

The project was designed around the following principles:

### Separation of Concerns

Frontend, backend, and database responsibilities are isolated to simplify maintenance and future enhancements.

### API-First Design

The backend exposes RESTful endpoints that can be consumed by multiple clients independently of the Angular application.

### Containerized Deployment

Docker support allows the application to run consistently across local, staging, and production environments.

### Scalability

The architecture supports future enhancements such as:

* Real-time telemetry streaming
* Historical telemetry analytics
* Authentication and authorization
* Alerting systems
* Multi-station deployments

## Local Development Setup

### Prerequisites

* Python 3.11+
* Node.js 20+
* PostgreSQL
* Angular CLI

---

### Backend

```bash
cd backend

python -m venv .venv

source .venv/bin/activate

pip install -r requirements.txt

uvicorn app.main:app --reload
```

Backend URL:

```text
http://localhost:8000
```

API Documentation:

```text
http://localhost:8000/docs
```

---

### Frontend

```bash
cd frontend

npm install

ng serve
```

Frontend URL:

```text
http://localhost:4200
```
